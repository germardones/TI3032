3. Extrayendo Datos: Read (R)

La lectura es la base de todo Dashboard y sistema analítico. En vez del inmenso bloque de texto plano de SELECT * FROM tabla WHERE ..., MongoDB utiliza un objeto JSON en JavaScript puro que actúa como un "molde" conceptual para atrapar la información deseada usando la función universal find().

Listar Todo: find() Vacío

El argumento de find() es el Filtro JSON. Si pasas un objeto literal vacío {}, recuperas el 100% absoluto de la colección.

> db.clientes.find( {} )   // Muestra todo el volumen almacenado

Filtros de Igualdad Estricta

Si buscas una coincidencia 1 a 1, solo inyecta los atributos que quieres forzar como llave y valor.

En mentalidad SQL:

SELECT * FROM clientes
WHERE nombre = 'Marta';

En Sintaxis MongoDB:

db.clientes.find({
  nombre: "Marta"
})

Uso de Operadores: Lógica con Dólares $

Cuando quieres filtrar por algo que NO es textual o exactamente igual (ej: mayores a 30, distintos de nulo), debes insertar un Operador Lógico Nativo que en MongoDB empieza obligatoriamente con el símbolo $.

La anatomía del objeto JSON cambia ligeramente anidando el operador:
{ atributo: { $operador: valor_a_evaluar } }

Operador Acrónimo Inglés Código de Búsqueda de Ejemplo
$eq / $ne EQual / Not Equal { edad: { $ne: 30 } }
(Filtra Usuarios que NO tengan 30 años)
$gt / $gte Greater Than (Equal) { edad: { $gte: 18 } }
(Filtra Mayores o iguales a 18 años)
$lt / $lte Less Than (Equal) { precio: { $lt: 500 } }
(Filtra Productos menores a $500 puros)
