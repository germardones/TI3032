1. Conceptos Base: El Ecosistema de MongoDB

Antes de escribir consultas, debemos entender en qué jerarquía estamos operando. En MongoDB, los datos se almacenan siguiendo una estructura de tres niveles lógicos descendentes.

1. Database (Base de Datos)

Instancia mayor. Al igual que en SQL, separa lógicamente universos de negocio enteros (ej: "GlobalMarketDB", "RRHH").

2. Colección

Equivalente directo a una Tabla en SQL. Agrupa un conjunto masivo de Documentos de un dominio semántico. (Ej: Colección Usuarios).

3. Documento

El equivalente a un Registro o Fila de DB. Es el bloque JSON físico definitivo que incrusta toda la información procesada.

Anatomía de un Comando en MongoDB Shell (Mongosh)

Cuando te conectas vía nube o consola local a MongoDB, envías comandos en formato de interfaz Javascript de objetos (C-Level). La estructura base y vital de la sintaxis obedece a un flujo estricto:

db.nombre_coleccion.comando_accion( { ...argumentos } )
  • 1. db (Contexto): Le indica imperativamente al motor "aplica la orden a la Base de Datos que estoy usando activamente".
  • 2. nombre_coleccion (Target): Apunta al modelo de grupo al que nos dirigimos (Ej: db.comentarios...).
  • 3. comando_accion() (El Filtro CRUD): La inyección de método (Ej: insertOne(), find(), deleteOne()).

El Comienzo Absoluto: Previo a escribir ningún comando a db, en tu Shell debes instruir la orden textual use nombre_de_mi_db para conectarte lógicamente a ese entorno de trabajo.
