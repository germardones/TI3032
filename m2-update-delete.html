4. Estructuras Avanzadas: Update y Delete

Modificar y eliminar datos son operaciones potencialmente destructivas de alto nivel. En MongoDB, al igual que SQL con su clausula algorítmica WHERE, la clave de no destruir la base entera reside en dominar de antemano el Filtro Inicial.

Actualización: updateOne() y updateMany()

A diferencia de Insertion, los comandos Update son agresivos y reciben siempre DOS objetos en secuencia, separados por una coma en el Core de la función:

  • Objeto JSON 1 (El Scope/Filtro): "¿A qué documentos/coordinadas atrapamos?"
  • Objeto JSON 2 (La Mutación): "¿Qué bloque de datos nuevo aplicamos sobrescribiendo?"

Si la empresa desea promover a "Marta" como usuario premium, la orden requiere empalmar un operador atómico y táctico: el $set.
Advertencia: Sin el operador $set inyectado en el paso 2, MongoDB borraría todos los atributos previos de Marta limpiando su Documento y colocando puramente el string nuevo.

> db.clientes.updateOne(
  { nombre: "Marta" },             // JSON 1: Filtro atrapador. Encuentra a Marta.
  { $set: { premium: true } }    // JSON 2: Mutación parcial atómica por $set
)

Borrado: deleteOne() y deleteMany()

La desvinculación es rápida pues no requiere Mutación y se ejecuta solo pasando el JSON Scope Filtro original. Un error mal enfocado equivale matemáticamente al temido DELETE FROM tabla sin el WHERE estricto.

Precaución Absoluta en Cascada

> db.clientes.deleteOne( { _id: ObjectId("64eb98f3b...") } )

Mejores Prácticas: En entornos en Producción se restringe el borrado en masa y siempre se destruye apuntando vectorialmente mediante la **Primary Key Pura** (`_id`) generada por BSON en vez de texto plano como el nombre. Ejecutar un deleteMany({ edad: { $gt: 50 } }) destruiría masivamente y sin reversión a una cuota poblacional completa.
