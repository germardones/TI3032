MÓDULO 03

1. Instalación de MongoDB

MongoDB se distribuye en tres capas de herramientas: el proceso daemon mongod que ejecuta el motor en disco, el shell cliente mongosh para interacción programática en REPL JavaScript, y MongoDB Compass como GUI de administración. Antes de instalar es clave entender la diferencia entre las ediciones disponibles.

Community Server vs Enterprise Server

MongoDB se distribuye en dos ediciones principales. Para entornos académicos y la mayoría de proyectos productivos, Community Server es más que suficiente.

Community Server (Gratuito)

  • Licencia SSPL (Server Side Public License) — gratuito para uso propio
  • Motor WiredTiger completo con compresión de datos
  • Replicación, sharding y transacciones ACID multi-documento
  • Aggregation pipeline, índices de texto, geoespaciales y TTL
  • Soporte oficial para Linux, macOS y Windows
  • El estándar en toda documentación y curso académico

Enterprise Server (Pago)

  • Licencia comercial — requiere contrato con MongoDB Inc.
  • Autenticación LDAP / Kerberos para integración corporativa
  • Motor de almacenamiento cifrado en reposo (WT Encrypted)
  • Auditoría detallada de operaciones (audit log)
  • Soporte oficial 24/7 con SLA garantizado
  • MongoDB Ops Manager para administración centralizada

Arquitectura de Componentes

mongod

Proceso servidor (daemon). Gestiona archivos de datos en disco con el motor WiredTiger. Escucha en el puerto TCP/27017. Es quien procesa lecturas y escrituras reales.

mongosh

Shell cliente basado en V8 (el motor JS de Node.js). Envía comandos al mongod vía protocolo de red. Sustituyó al antiguo mongo shell desde MongoDB 5.0.

Compass

GUI oficial. Permite explorar colecciones, construir queries visualmente, analizar índices y revisar planes de ejecución (explain plans) sin escribir comandos manualmente.

Instalación en Linux — Ubuntu 22.04 / 24.04

El método correcto en Linux es usar el repositorio oficial de MongoDB vía APT. Los paquetes de los repositorios del sistema operativo suelen estar desactualizados varios ciclos de versión. El proceso requiere importar la clave GPG pública para verificar la autenticidad de los paquetes.

Instalación completa paso a paso en Ubuntu

# Paso 1: Importar la clave GPG pública de MongoDB 7.x
curl -fsSL https://www.mongodb.org/static/pgp/server-7.0.asc | \
  sudo gpg -o /usr/share/keyrings/mongodb-server-7.0.gpg --dearmor

# Paso 2: Registrar el repositorio oficial en APT
echo "deb [ arch=amd64,arm64 \
  signed-by=/usr/share/keyrings/mongodb-server-7.0.gpg ] \
  https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | \
  sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list

# Paso 3: Actualizar índice e instalar el engine
sudo apt-get update
sudo apt-get install -y mongodb-org

# Paso 4: Iniciar el daemon y habilitarlo en el arranque del OS
sudo systemctl start mongod
sudo systemctl enable mongod

# Confirmar estado del proceso del daemon
sudo systemctl status mongod

Los archivos de datos del motor WiredTiger se almacenan en /var/lib/mongodb/ y los logs del daemon en /var/log/mongodb/mongod.log. La configuración del servidor reside en /etc/mongod.conf en formato YAML.

Instalación en macOS — Homebrew

macOS no distribuye MongoDB en sus repositorios nativos. Se usa el tap oficial de Homebrew mantenido por MongoDB Inc. Requiere tener Homebrew instalado previamente.

# Registrar el tap oficial de fórmulas MongoDB
brew tap mongodb/brew

# Instalar MongoDB Community Server 7.0
brew install mongodb-community@7.0

# Iniciar el servicio via launchd (auto-reinicio tras reboot)
brew services start mongodb-community@7.0

Alternativa con Docker — Entornos de Desarrollo Modernos

Docker es el estándar en entornos profesionales y CI/CD. Permite ejecutar múltiples versiones de MongoDB en paralelo en contenedores aislados sin conflictos de dependencias en el sistema anfitrión.

# Descargar imagen oficial y levantar el contenedor con persistencia
docker run -d \
  --name mongodb-dev \
  -p 27017:27017 \
  -e MONGO_INITDB_ROOT_USERNAME=admin \
  -e MONGO_INITDB_ROOT_PASSWORD=secreto \
  -v mongodb_data:/data/db \
  mongo:7.0

# Conectar el shell directamente al contenedor
docker exec -it mongodb-dev mongosh \
  -u admin -p secreto --authenticationDatabase admin

*El flag -v mongodb_data:/data/db crea un volumen Docker persistente. Sin él, todos los datos se destruyen cuando el contenedor se elimina. -p 27017:27017 expone el puerto del contenedor al sistema anfitrión para que mongosh local pueda conectar.*

Alternativa Cloud — MongoDB Atlas

MongoDB Atlas es el servicio gestionado oficial (DBaaS). Ofrece un clúster gratuito permanente M0 de 512 MB desplegado en AWS, GCP o Azure. Es la opción ideal para proyectos académicos y prototipos donde no se requiere infraestructura local.

URI de Conexión Atlas: Todos los clientes (mongosh, drivers de Node.js, Python, Java) se conectan mediante una cadena SRV:

mongodb+srv://usuario:contraseña@cluster0.abc.mongodb.net/miBaseDeDatos

Este string incluye descubrimiento automático de nodos del clúster vía DNS SRV, reintento automático de escrituras y TLS habilitado por defecto.

Verificación de la Instalación

Ejecuta estos comandos de diagnóstico en mongosh para confirmar que el motor responde correctamente:

# Conectar al servidor local (puerto 27017 por defecto)
mongosh

# Consultar la versión del motor del servidor
db.version()               // → "7.0.x"

# Listar las bases de datos existentes en el servidor
show dbs

# Inspeccionar uptime del servidor (segundos activo)
db.serverStatus().uptime

Output esperado de show dbs: Verás tres bases de datos internas: admin (control de acceso y autenticación), config (metadatos de sharding) y local (datos de replicación no distribuidos). Estas son gestionadas exclusivamente por el motor. Nunca insertes datos directamente en ellas.
