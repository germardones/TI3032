4. Manipulación de Subdocumentos y Arreglos

Saber incrustar datos es solo la mitad del camino: el desafío real es consultarlos y modificarlos eficientemente una vez que viven anidados en el árbol de un documento. MongoDB ofrece una sintaxis de dot notation para navegar el árbol de objetos y un conjunto de operadores de arreglos que permiten modificar colecciones de forma atómica sin necesidad de reemplazar el documento completo.

Dot Notation — Navegando Estructuras Anidadas

La notación de punto (dot notation) permite referenciar campos de subdocumentos en queries y actualizaciones usando el formato "padre.hijo.nieto". Es el equivalente al operador . de JavaScript para acceder a propiedades de objetos anidados.

Consultar Subdocumentos

# Buscar usuarios de Santiago
# (campo "ciudad" dentro de "direccion")
db.usuarios.find({
  "direccion.ciudad": "Santiago"
})

# Filtro múltiple en subdocumento
db.usuarios.find({
  "direccion.region":  "Metropolitana",
  "perfil.nivel":      "senior"
})

Actualizar Subdocumentos

# Modificar un campo específico
# dentro del subdocumento embebido
db.usuarios.updateOne(
  { email: "vrios@example.com" },
  { $set: {
    "direccion.ciudad":        "Valparaíso",
    "direccion.codigo_postal": "2340000"
  }}
)

Por qué comillas: En mongosh las claves con dot notation deben ir entre comillas ("direccion.ciudad") porque el punto es un carácter especial en JavaScript. Sin comillas, el motor interpretaría direccion.ciudad como acceso a propiedad del objeto direccion, no como una ruta de campo MongoDB.

Consultando Arreglos — Operadores de Búsqueda

MongoDB ofrece operadores especializados para filtrar documentos en función del contenido de sus arreglos embebidos.

# $in — documentos donde el arreglo contiene AL MENOS uno de los valores
db.estudiantes.find({
  cursos_inscritos: { $in: ["curso_bd_nosql", "curso_docker"] }
})

# $all — documentos donde el arreglo contiene TODOS los valores
db.estudiantes.find({
  cursos_inscritos: { $all: ["curso_bd_nosql", "curso_python"] }
})

# $size — documentos donde el arreglo tiene exactamente N elementos
db.pedidos.find({
  items: { $size: 3 }   // Pedidos con exactamente 3 ítems
})

$elemMatch — Condiciones Compuestas en Arreglos

Cuando necesitas que un único elemento del arreglo cumpla simultáneamente múltiples condiciones, debes usar $elemMatch. Sin él, MongoDB evaluaría cada condición por separado contra diferentes elementos del arreglo.

Sin $elemMatch (incorrecto)

# Busca pedidos donde items tenga
# algún elemento con cantidad > 1
# Y algún elemento con precio < 50000
# (pueden ser elementos DISTINTOS!)
db.pedidos.find({
  "items.cantidad":        { $gt: 1 },
  "items.precio_unitario": { $lt: 50000 }
})

Con $elemMatch (correcto)

# Busca pedidos donde existe UN item
# que cumpla AMBAS condiciones a la vez
db.pedidos.find({
  items: {
    $elemMatch: {
      cantidad:        { $gt: 1 },
      precio_unitario: { $lt: 50000 }
    }
  }
})

Operadores de Mutación de Arreglos

Estos operadores modifican los arreglos de forma atómica — el motor aplica la mutación en una sola operación de escritura sin necesidad de leer el documento primero en el cliente, modificarlo en memoria y reescribirlo completo.

$push — Agregar Elementos al Arreglo

# Agregar un nuevo ítem al arreglo "items" de un pedido
db.pedidos.updateOne(
  { _id: ObjectId("507f191e810c19729de860ea") },
  { $push: {
    items: {
      producto_nombre: "Alfombrilla Gaming",
      cantidad:        1,
      precio_unitario: 15900
    }
  }}
)

# $each — insertar múltiples elementos en un solo $push
db.estudiantes.updateOne(
  { _id: ObjectId("est001") },
  { $push: {
    cursos_inscritos: {
      $each: [ObjectId("curso_devops"), ObjectId("curso_linux")]
    }
  }}
)

$pull — Eliminar Elementos del Arreglo

# Remover un curso específico del arreglo de cursos inscritos
db.estudiantes.updateOne(
  { _id: ObjectId("est001") },
  { $pull: { cursos_inscritos: ObjectId("curso_python") } }
)

# $pull con condición — remover todos los ítems con precio < 20000
db.pedidos.updateOne(
  { _id: ObjectId("507f191e810c19729de860ea") },
  { $pull: {
    items: { precio_unitario: { $lt: 20000 } }
  }}
)

$addToSet — Agregar Sólo si No Existe

# Agregar un tag solo si no está duplicado en el arreglo
# (se comporta como un Set matemático — sin duplicados)
db.articulos.updateOne(
  { _id: ObjectId("art001") },
  { $addToSet: { tags: "mongodb" } }
)
// Si "mongodb" ya existe en el arreglo → no hace nada
// Si no existe → lo agrega. Sin duplicados garantizados.

$pop — Remover Primer o Último Elemento

db.cola_tareas.updateOne(
  { _id: ObjectId("cola001") },
  { $pop: { tareas: -1 } }  // -1 elimina el primer elemento (FIFO)
)

db.historial.updateOne(
  { _id: ObjectId("hist001") },
  { $pop: { eventos: 1 } }   // +1 elimina el último elemento (LIFO)
)

El Operador Posicional $ — Actualizar un Elemento Encontrado

Cuando necesitas actualizar el primer elemento de un arreglo que cumple el filtro de la query, sin conocer su índice numérico, usas el operador posicional $. Funciona como un puntero al elemento que hizo match en el filtro.

# Actualizar la cantidad del ítem "Teclado Mecánico" en el pedido
# El $ referencia automáticamente el elemento del arreglo que hizo match
db.pedidos.updateOne(
  {
    _id:                   ObjectId("507f191e810c19729de860ea"),
    "items.producto_nombre": "Teclado Mecánico"  // Condición que activa el $
  },
  {
    $set: { "items.$.cantidad": 2 }  // $ apunta al ítem que hizo match arriba
  }
)

Resumen de operadores de arreglos: $push agrega (con $each para múltiples), $pull elimina por valor o condición, $addToSet agrega sin duplicar, $pop extrae por posición, y el operador posicional $ actualiza el elemento que cumplió el filtro. Juntos forman el toolkit completo para manipular arrays de forma atómica en MongoDB.
