3. Agrupación y Desanidado: $group y $unwind

Si $match reduce y $project moldea, $group y $unwind son las etapas que realmente transforman la estructura de los datos. Juntas permiten convertir colecciones de documentos complejos en resúmenes analíticos.

$group — Agrupar y acumular

$group es el equivalente de GROUP BY en SQL. Colapsa múltiples documentos en uno por cada valor único del campo _id, aplicando acumuladores para calcular métricas.

// Estructura de $group
{ $group: {
    _id: <expresión de agrupación>,   // campo(s) por los que agrupar
    campo1: { $acumulador1: <expr> },
    campo2: { $acumulador2: <expr> }
}}

Acumuladores principales

Acumulador Qué hace Equivalente SQL
$sumSuma. Con 1 cuenta documentos.SUM / COUNT
$avgPromedio del campoAVG
$min / $maxValor mínimo / máximoMIN / MAX
$pushAcumula valores en un arreglo (con duplicados)GROUP_CONCAT
$addToSetAcumula valores únicos (sin duplicados)DISTINCT
$first / $lastPrimer / último valor del grupoFIRST_VALUE / LAST_VALUE
$countCuenta documentos del grupo (desde MongoDB 5.0)COUNT(*)

Ejemplos de $group

// 1. Contar pedidos y sumar ventas por ciudad
db.pedidos.aggregate([
  {
    $group: {
      _id:           "$ciudad",
      total_pedidos: { $sum: 1 },
      ingresos:      { $sum: "$monto" },
      ticket_prom:   { $avg: "$monto" }
    }
  }
])

/* Resultado:
[
  { _id: "Santiago",   total_pedidos: 84, ingresos: 12400000, ticket_prom: 147619 },
  { _id: "Valparaíso", total_pedidos: 31, ingresos:  4200000, ticket_prom: 135483 }
] */

// 2. Agrupar por múltiples campos (ciudad + mes)
db.pedidos.aggregate([
  {
    $group: {
      _id: {
        ciudad: "$ciudad",
        mes:    { $month: "$fecha" }
      },
      total: { $sum: "$monto" }
    }
  }
])

// 3. _id: null → agrupa TODO en un solo documento
db.pedidos.aggregate([
  {
    $group: {
      _id:        null,
      gran_total: { $sum: "$monto" },
      cantidad:   { $sum: 1 }
    }
  }
])

$unwind — Desanidar arreglos

Cuando un documento tiene un campo que es un arreglo, $unwind lo explota: genera un documento nuevo por cada elemento del arreglo, copiando el resto del documento original. Es indispensable antes de agrupar por elementos de un array.

Antes de $unwind
{
  pedido_id: "P001",
  cliente:   "Ana",
  items: [
    "Teclado",
    "Mouse",
    "Monitor"
  ]
}
// 1 documento con 3 elementos
→
Después de $unwind: "items"
{ pedido_id:"P001", cliente:"Ana", items:"Teclado"  }
{ pedido_id:"P001", cliente:"Ana", items:"Mouse"    }
{ pedido_id:"P001", cliente:"Ana", items:"Monitor"  }
// 3 documentos — uno por elemento
// Sintaxis básica
db.pedidos.aggregate([
  { $unwind: "$items" }   // el campo debe comenzar con $
])

// Sintaxis extendida — preservar documentos con arreglo vacío o null
db.pedidos.aggregate([
  {
    $unwind: {
      path:                       "$items",
      preserveNullAndEmptyArrays: true   // no elimina docs sin items
    }
  }
])

El patrón estrella: $unwind + $group

El caso de uso más frecuente: tienes un arreglo embebido (items de un pedido, tags de un artículo, recetas de una cita) y quieres obtener estadísticas sobre sus elementos.

// Top 5 productos más vendidos (items embebidos en pedidos)
db.pedidos.aggregate([
  {
    $match: { estado: "entregado" }        // 1. solo pedidos entregados
  },
  {
    $unwind: "$items"                      // 2. un doc por cada ítem
  },
  {
    $group: {
      _id:      "$items.nombre_producto",  // 3. agrupar por producto
      vendidos: { $sum: "$items.cantidad" },
      ingresos: { $sum: {
        $multiply: ["$items.cantidad", "$items.precio_unitario"]
      }}
    }
  },
  {
    $sort: { vendidos: -1 }               // 4. de mayor a menor
  },
  {
    $limit: 5                             // 5. solo los 5 primeros
  }
])

/* Resultado esperado:
[
  { _id: "Teclado Mecánico",  vendidos: 234, ingresos: 19890000 },
  { _id: "Mouse Inalámbrico", vendidos: 198, ingresos:  4059900 },
  { _id: "Monitor 27\"",      vendidos:  87, ingresos: 43413000 },
  ...
] */

Atención: $unwind elimina por defecto los documentos cuyo campo de arreglo es null, no existe, o el arreglo está vacío. Si necesitas conservarlos, usa preserveNullAndEmptyArrays: true. Esto es especialmente importante al migrar datos desde SQL donde algunos campos pueden estar vacíos.
