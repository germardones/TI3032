1. Introducción al Aggregation Pipeline

El Aggregation Pipeline es el motor de análisis de MongoDB: una tubería de etapas encadenadas donde cada etapa transforma la colección de documentos y pasa su resultado a la siguiente. A diferencia de find(), que solo filtra y proyecta, el pipeline puede agrupar, calcular, desanidar, ordenar y combinar datos de forma declarativa.

¿Por qué existe el pipeline?

SQL — Análisis con GROUP BY

  • Filtra con WHERE
  • Agrupa con GROUP BY
  • Calcula con funciones de agregación (SUM, COUNT…)
  • Ordena con ORDER BY
  • Todo en una sola sentencia declarativa

MongoDB — Aggregation Pipeline

  • Filtra con $match
  • Agrupa con $group
  • Calcula con acumuladores ($sum, $avg…)
  • Ordena con $sort
  • Cada operación es una etapa independiente del array

Anatomía de un Pipeline

Un pipeline es un array de objetos. Cada objeto representa una etapa con exactamente un operador ($match, $group, etc.). Los documentos fluyen de izquierda a derecha, transformándose en cada paso.

// Sintaxis general
db.coleccion.aggregate([
  { $etapa1: { ... } },   // transforma los docs
  { $etapa2: { ... } },   // recibe los docs de $etapa1
  { $etapa3: { ... } }    // recibe los docs de $etapa2
])

Ejemplo mínimo: contar pedidos por estado

Supón que tienes una colección pedidos. Queremos saber cuántos pedidos hay en cada estado (pendiente, entregado, cancelado).

// Pipeline de 2 etapas
db.pedidos.aggregate([
  {
    $group: {
      _id:   "$estado",        // campo que agrupa
      total: { $sum: 1 }       // cuenta documentos
    }
  },
  {
    $sort: { total: -1 }       // de mayor a menor
  }
])

/* Resultado esperado:
[
  { _id: "entregado", total: 142 },
  { _id: "pendiente", total:  38 },
  { _id: "cancelado", total:  17 }
] */

Las etapas más importantes

$match

Filtra documentos (como find()). Siempre ponlo al principio para reducir la carga de trabajo del pipeline.

$group

Agrupa documentos por un campo y aplica acumuladores: $sum, $avg, $min, $max, $push.

$project

Define qué campos incluir, excluir o renombrar en la salida. También permite crear campos calculados.

$unwind

Desanida un arreglo: convierte cada elemento en un documento separado. Imprescindible antes de agrupar por elementos de arrays.

$sort

Ordena los documentos. 1 = ascendente, -1 = descendente. Úsalo antes de $limit para obtener los top-N.

$limit / $skip

$limit restringe la cantidad de documentos de salida. $skip salta los primeros N. Juntos implementan paginación.

Flujo visual de datos

Imagina el pipeline como una cadena de montaje: cada etapa recibe documentos crudos y entrega documentos transformados a la siguiente.

Regla de oro: Coloca siempre $match al inicio del pipeline. MongoDB puede usar índices para ese filtro antes de procesar los documentos, lo que reduce drásticamente la carga. Un $match al final requiere procesar todos los documentos primero.
