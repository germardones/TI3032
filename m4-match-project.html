MÓDULO 04

2. Filtrado y Proyección: $match y $project

Estas dos etapas son el punto de entrada de casi todo pipeline: $match reduce el volumen de datos al mínimo necesario y $project moldea la forma de cada documento antes de entregarlo a las etapas siguientes.

$match — Filtrar documentos

$match usa exactamente la misma sintaxis de filtros que find(): operadores de comparación, lógicos, regex, $elemMatch, etc. La diferencia es que aquí forma parte de un pipeline encadenado.

// Colección: ventas
// Objetivo: ventas de 2024 con monto > 500 000
db.ventas.aggregate([
  {
    $match: {
      fecha:  { $gte: new Date("2024-01-01"),
                $lt:  new Date("2025-01-01") },
      monto:  { $gt: 500000 },
      estado: "pagado"
    }
  }
])

Rendimiento: cuando $match es la primera etapa, MongoDB puede usar índices de la colección. Si lo colocas después de $group o $project, ya no hay índice disponible — el motor debe escanear todos los documentos intermedios.

$match con operadores lógicos

// Productos de electrónica O con stock bajo
db.productos.aggregate([
  {
    $match: {
      $or: [
        { categoria: "electronica" },
        { stock: { $lt: 10 } }
      ]
    }
  }
])

// Clientes de Santiago que NO tienen deuda
db.clientes.aggregate([
  {
    $match: {
      ciudad:    "Santiago",
      deuda:     { $not: { $gt: 0 } }
    }
  }
])

$project — Dar forma a los documentos

$project controla qué campos aparecen en la salida de esa etapa. Con 1 incluyes, con 0 excluyes. Además puedes crear campos nuevos con expresiones o renombrar existentes.

Incluir campos (1)

# Solo nombre, email y ciudad.
# _id se incluye por defecto;
# para excluirlo usa _id: 0
db.clientes.aggregate([
  {
    $project: {
      nombre: 1,
      email:  1,
      ciudad: 1,
      _id:    0
    }
  }
])

Excluir campos (0)

# Todo excepto password y __v
# No mezcles 0 y 1 (salvo _id)
db.clientes.aggregate([
  {
    $project: {
      password: 0,
      __v:      0
    }
  }
])

Campos calculados con expresiones

$project puede crear campos nuevos usando operadores de expresión del pipeline. Es el equivalente de AS campo_calculado en SQL.

// Calcular precio con IVA y formatear nombre completo
db.productos.aggregate([
  {
    $project: {
      nombre:       1,
      precio_neto:  "$precio",
      precio_iva: {
        $multiply: ["$precio", 1.19]   // campo calculado
      },
      categoria:    { $toUpper: "$categoria" }
    }
  }
])

// Concatenar nombre completo desde dos campos
db.empleados.aggregate([
  {
    $project: {
      nombre_completo: {
        $concat: ["$nombre", " ", "$apellido"]
      },
      anio_ingreso: {
        $year: "$fecha_ingreso"          // extrae el año
      }
    }
  }
])

Combinando $match + $project

El patrón más común: primero filtrar para reducir el conjunto, luego proyectar solo los campos necesarios para las etapas siguientes.

// Ventas aprobadas en 2024 → solo cliente, monto y mes
db.ventas.aggregate([
  {
    $match: {
      estado: "aprobado",
      fecha: { $gte: new Date("2024-01-01") }
    }
  },
  {
    $project: {
      _id:      0,
      cliente:  "$nombre_cliente",
      monto:    1,
      mes: {
        $month: "$fecha"               // número de mes (1-12)
      }
    }
  }
])

/* Resultado esperado:
[
  { cliente: "Ana López",   monto: 780000, mes: 3 },
  { cliente: "Juan Pérez",  monto: 540000, mes: 5 },
  ...
] */

Operadores de expresión más usados en $project

Operador Descripción Ejemplo
$multiply Multiplica valores { $multiply: ["$precio", 1.19] }
$add / $subtract Suma / resta { $subtract: ["$precio", "$descuento"] }
$concat Concatena strings { $concat: ["$nombre", " ", "$apellido"] }
$toUpper / $toLower Cambia mayúsculas/minúsculas { $toUpper: "$categoria" }
$year / $month / $dayOfMonth Extrae partes de una fecha { $year: "$fecha_ingreso" }
$cond IF-THEN-ELSE dentro del pipeline { $cond: [{ $gt: ["$stock", 0] }, "disponible", "agotado"] }
$size Longitud de un arreglo { $size: "$items" }

Diferencia clave con find(): find() solo puede incluir o excluir campos existentes. $project dentro del pipeline puede crear campos nuevos, aplicar funciones matemáticas, de cadena o de fecha, y renombrar campos — algo imposible con find().
