MÓDULO 04

4. $sort, $limit y Pipelines Completos

Las etapas $sort, $limit y $skip son las herramientas de presentación del pipeline: ordenan, recortan y paginan los resultados. En esta sección también veremos cómo combinar todas las etapas aprendidas para construir pipelines analíticos reales.

$sort — Ordenar resultados

Funciona igual que el .sort() de find(): 1 = ascendente, -1 = descendente. Puede ordenar por múltiples campos.

// Ordenar por un campo
db.productos.aggregate([
  { $sort: { precio: -1 } }          // de mayor a menor precio
])

// Ordenar por múltiples campos
db.empleados.aggregate([
  {
    $sort: {
      departamento: 1,               // primero por depto (A→Z)
      salario:      -1               // luego por salario (mayor→menor)
    }
  }
])

// $sort después de $group (campo calculado)
db.ventas.aggregate([
  {
    $group: {
      _id:   "$region",
      total: { $sum: "$monto" }
    }
  },
  { $sort: { total: -1 } }           // ranking de regiones
])

Rendimiento: si $sort es la primera etapa y el campo tiene índice, MongoDB usa el índice para ordenar sin cargar los documentos en memoria. Después de $group no hay índice disponible — el motor ordena en memoria, lo cual puede ser costoso con millones de grupos.

$limit y $skip — Recortar y paginar

$limit — Top N

# Equivale a LIMIT en SQL.
# Detiene el pipeline después
# de N documentos.
db.productos.aggregate([
  { $sort:  { ventas: -1 } },
  { $limit: 10 }             // top 10
])

$skip — Paginación

# Salta los primeros N documentos.
# $sort → $skip → $limit = página.
db.productos.aggregate([
  { $sort:  { nombre: 1 } },
  { $skip:  20 },            // página 3
  { $limit: 10 }             // tamaño 10
])

Etapa adicional: $count

Reemplaza todos los documentos de la etapa anterior por un único documento con el conteo total. Útil al final de un pipeline de filtrado.

// Cuántos pedidos superan los $200 000
db.pedidos.aggregate([
  { $match:  { monto: { $gt: 200000 } } },
  { $count:  "pedidos_grandes" }
])
// Resultado: [ { pedidos_grandes: 47 } ]

Pipelines completos — Casos reales

Ahora combinamos todo lo aprendido. Estos tres pipelines son exactamente el tipo de análisis que se pide en el mini proyecto.

Pipeline 1 — Ranking de médicos por especialidad

// Colección: citas (con médico embebido o referenciado)
// Objetivo: cantidad de citas realizadas por especialidad
db.citas.aggregate([
  {
    $match: { estado: "REALIZADA" }          // solo citas completadas
  },
  {
    $group: {
      _id: {
        especialidad: "$medico.especialidad",
        medico:       "$medico.nombre"
      },
      total_citas: { $sum: 1 }
    }
  },
  {
    $sort: { total_citas: -1 }               // de mayor a menor
  },
  {
    $project: {
      _id:         0,
      especialidad: "$_id.especialidad",
      medico:       "$_id.medico",
      total_citas:  1
    }
  }
])

Pipeline 2 — Top 5 medicamentos más recetados

// Colección: citas (con recetas embebidas en cada cita)
// Cada cita tiene un arreglo "recetas" con subdocumentos
db.citas.aggregate([
  {
    $match: { estado: "REALIZADA" }
  },
  {
    $unwind: "$recetas"                      // un doc por cada receta
  },
  {
    $group: {
      _id:   "$recetas.medicamento",
      veces: { $sum: 1 },
      duracion_promedio: { $avg: "$recetas.duracion_dias" }
    }
  },
  {
    $sort: { veces: -1 }
  },
  {
    $limit: 5                                // solo los 5 primeros
  },
  {
    $project: {
      _id:              0,
      medicamento:      "$_id",
      veces_recetado:   "$veces",
      duracion_prom_dias: { $round: ["$duracion_promedio", 1] }
    }
  }
])

/* Resultado esperado:
[
  { medicamento: "Ibuprofeno 400mg",   veces_recetado: 38, duracion_prom_dias: 7.2 },
  { medicamento: "Amoxicilina 500mg",  veces_recetado: 29, duracion_prom_dias: 10.0 },
  { medicamento: "Paracetamol 500mg",  veces_recetado: 27, duracion_prom_dias: 5.5 },
  ...
] */

Pipeline 3 — Carga mensual de citas

// Objetivo: número de citas agrupadas por mes y año
db.citas.aggregate([
  {
    $group: {
      _id: {
        anio: { $year:  "$fecha" },
        mes:  { $month: "$fecha" }
      },
      total_citas:     { $sum: 1 },
      citas_realizadas: {
        $sum: { $cond: [{ $eq: ["$estado", "REALIZADA"] }, 1, 0] }
      },
      citas_canceladas: {
        $sum: { $cond: [{ $eq: ["$estado", "CANCELADA"] }, 1, 0] }
      }
    }
  },
  {
    $sort: {
      "_id.anio": 1,
      "_id.mes":  1
    }
  },
  {
    $project: {
      _id:              0,
      periodo:          { $concat: [
        { $toString: "$_id.anio" }, "-",
        { $toString: "$_id.mes"  }
      ]},
      total_citas:      1,
      citas_realizadas: 1,
      citas_canceladas: 1
    }
  }
])

/* Resultado esperado:
[
  { periodo: "2024-1", total_citas: 12, citas_realizadas: 10, citas_canceladas: 2 },
  { periodo: "2024-2", total_citas: 18, citas_realizadas: 15, citas_canceladas: 3 },
  ...
] */

Resumen del Módulo 4

$match

Filtra al inicio. Usa índices. Reduce la carga del pipeline.

$project

Moldea la salida. Crea campos calculados. Limpia el documento.

$group

Agrupa con acumuladores. Equivale a GROUP BY + funciones de agregación.

$unwind

Explota arreglos. Necesario antes de agrupar por elementos de arrays.

$sort

Ordena. Al inicio usa índices; después de $group ordena en memoria.

$limit / $skip

Recorta y pagina. Siempre después de $sort para resultados deterministas.

Patrón de diseño del pipeline: $match → $unwind (si hay arrays) → $group → $sort → $limit → $project. Este orden minimiza el volumen de datos en cada etapa y produce el pipeline más eficiente posible.
